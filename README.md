# 末泠的个人网站

纯像素风个人网站，主页只保留 Mod、工具、登录三个入口。Mod 和工具页面保留内置内容，同时会叠加显示后台上传的新内容。

## 页面结构

- `index.html`：主页，只保留 `Mod`、`工具`、`登录` 三个入口按钮
- `mods.html`：Mod 展柜，显示内置 Mod 和后台上传的 Mod，支持标签、分页、详情、更新日志、版本历史、文件校验、下载统计和截图放大
- `tools.html`：工具库，显示内置工具和后台上传的工具，保持极简展示，只含搜索、分页、详情、下载和下载统计
- `admin.html`：后台，使用管理员密码 + GitHub Token 登录，可上传 Mod 或工具并同步到 GitHub 仓库
- `script.js`：前端数据渲染、下载统计、灯箱、后台上传、读取后台同步数据
- `styles.css`：纯像素风 UI，中文字体使用 `assets/Text.ttf`，并恢复 `assets/background.gif` 背景
- `server.js`：Node.js 后端，提供管理员登录、下载统计、上传和 GitHub 仓库同步接口
- `package.json`：Node.js 启动配置

## 后端接口

- `POST /api/auth/login`：管理员密码 + GitHub Token 登录
- `GET /api/me`：当前登录状态
- `GET /api/auth/logout`：退出登录
- `GET /api/data`：读取下载统计和上传记录
- `POST /api/download`：记录下载次数
- `POST /api/admin/upload`：上传 Mod 或工具并同步到 GitHub 仓库

## 上传规则

- 前台保留内置 Mod / 工具，后台上传的新内容会追加显示
- Mod 允许预设标签和自定义标签
- 工具不显示分类和标签
- 源码下载按钮只在登录后显示

## 环境变量

部署前需要设置：

- `BASE_URL`：网站线上地址，例如 `https://example.com`
- `GITHUB_OWNER`：GitHub 仓库所有者
- `GITHUB_REPO`：GitHub 仓库名
- `GITHUB_BRANCH`：分支名，默认 `main`
- `ADMIN_PASSWORD`：管理员密码，默认 `admin123`
- `PORT`：端口，默认 `3000`

GitHub Token 在后台登录时输入，用于上传文件和同步仓库数据。

## 本地运行

```bash
npm start
```

然后打开 `http://localhost:3000`。
